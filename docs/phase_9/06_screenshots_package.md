# Phase 9 Screenshots Package

Screenshots were captured during the dashboard/productization phase and should be attached to the final GitHub or portfolio submission. This document defines the recommended screenshot package without modifying the application.

## Recommended Screenshot Files

| File Name | Screen | Purpose |
|---|---|---|
| `01_executive_dashboard.png` | Dashboard | Shows executive and operational metrics |
| `02_risk_queue.png` | Risk Queue | Shows prioritized recommendations and filters |
| `03_case_detail.png` | Case Detail | Shows recommendation, evidence, context, and relationship mapping |
| `04_agent_timeline.png` | Case Detail timeline section | Shows agent execution history |
| `05_governance_or_audit.png` | Governance/API/audit view if captured | Shows auditability and governance readiness |

## Screenshot Checklist

- Use synthetic data only.
- Avoid showing local secrets, environment variables, terminal tokens, or browser account data.
- Show `CLM2001` as the primary demo case.
- Capture one screenshot that makes the product value clear without live explanation.
- Capture one screenshot that proves agent explainability and auditability.

## Submission Guidance

Attach screenshots to the final portfolio post or GitHub discussion. Keep the repository documentation stable by referencing screenshot names here rather than embedding large image binaries unless a final asset folder is explicitly requested.
