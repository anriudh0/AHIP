Controlled LLM Enhancement

One focused enhancement point for clearer explanations.

The Consolidator uses an LLM only to refine wording; it does not change decision logic.

Why only one node?

Safety

Limiting LLM access to the Consolidator keeps routing, scoring and audit deterministic. The LLM receives a minimal, redacted context and a strict prompt contract.

What the LLM can and cannot do

Boundaries
  • Can refine language, create an executive summary, and improve readability.
  • Cannot change route selection, risk scores, protected fields, or audit records.
  • All responses must pass schema validation before display.

The healthcare challenge

Fragmented data & manual review
  • Information is spread across patients, providers, benefit plans, claims records, care history and contracts.

  • Review teams need clear, auditable narratives that do not change the underlying business logic.

AHIP unifies evidence from specialist agents, preserves deterministic business rules, and delivers explainable recommendations to reviewers.

Implementation

These are the pieces that enable the controlled LLM enhancement.

Expand a card to read implementation details.

It isolates prompts, schemas, provider resolution, service logic, and validation so the Consolidator can call the LLM without leaking responsibilities into the graph nodes.

The prompt contract tells the LLM what to generate, what not to touch, and how to stay inside the structured JSON format required by the validator.

The adapter keeps provider resolution behind one interface, which makes the system easy to swap, test, and protect from circular dependencies.

It behaves like a provider, but returns consistent structured content so the pipeline can be tested without external calls.

Structured validation checks the output format, the response fields, and the protected-field contract before anything is shown to the user.

The service wraps context creation, provider execution, validation, and fallback so the rest of the application only sees a reliable result.

The Consolidator receives deterministic outputs, calls the LLM only for narrative enhancement, and preserves the original recommendation logic.

The response always carries enough metadata to explain what happened, what failed, and whether fallback was used.
